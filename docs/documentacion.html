🔷 OGA Suite — Backend FastAPI

Documentación técnica completa del backend profesional construido para el Banco de Guayaquil. Reemplaza 3 endpoints genéricos SQL-injection-prone por 42 endpoints REST semánticos con autenticación Azure AD y control de permisos OGA basado en base de datos.

🗄️ Base de Datos

  • Motor: MySQL
  • Host: 172.26.61.35:3323
  • BD: BG_OGASUITE
  • Usuario: apl_ogasuite
  • Driver: PyMySQL + DictCursor
  • Pool: 10 conexiones thread-safe

🔐 Autenticación

  • Proveedor: Azure Entra ID
  • Protocolo: Bearer JWT (RS256)
  • TenantID: d5cb91b6-...
  • ClientID: 893b18cb-...
  • JWKS: cache 24h automático
  • OGA: tabla t_oga_usuarios

⚡ Performance

  • FastAPI 0.115 + Uvicorn
  • Árbol metadatos: cache 15min
  • Permisos usuario: cache 5min
  • Términos: cache 10min
  • Stats dominio: asyncio paralelo
  • CORS: orígenes configurables
Problema resuelto: El frontend anterior enviaba SQL dinámico completo al servidor (POST /query con SELECT...). Ahora el backend expone endpoints semánticos REST — el frontend solo manda datos, nunca SQL. Esto elimina vulnerabilidades de SQL injection y centraliza toda la lógica de negocio.

El campo is_oga se determina consultando la tabla t_oga_usuarios en BD (no hardcodeado en código). Se cachea por 5 minutos por email.

Capa Core — Explicación detallada

📄 main.py — Punto de entrada

Crea la aplicación FastAPI con lifespan (reemplaza on_startup/on_shutdown). Al arrancar: inicializa el pool MySQL y configura el logging. Al apagar: cierra todas las conexiones del pool.

Configura CORS: solo acepta peticiones desde localhost:5173, localhost:5175, y gobinfoana01-2:3000. Métodos permitidos: GET, POST, PUT, DELETE. Headers: Authorization, Content-Type.

Registra los 8 routers bajo sus prefijos y registra los manejadores globales de excepciones.

📄 core/config.py — Configuración

Usa pydantic-settings para leer variables del archivo .env. Cached con @lru_cache para no releer el disco en cada petición. Variables clave:

VariableValor por defectoUso
SQL_HOST172.26.61.35IP del servidor MySQL
SQL_PORT3323Puerto MySQL (no estándar)
SQL_DATABASEBG_OGASUITEBase de datos
SQL_POOL_SIZE10Conexiones máximas en pool
AZURE_TENANT_IDd5cb91b6-...Para construir URL JWKS
AZURE_CLIENT_ID893b18cb-...Validar audience del JWT
CACHE_TTL_ARBOL_SEGUNDOS900TTL árbol metadatos (15min)
CACHE_TTL_PERMISOS_SEGUNDOS300TTL permisos usuario (5min)

📄 core/database.py — Pool de conexiones MySQL

Implementa un pool thread-safe con Semaphore (limita concurrencia a pool_size) + deque (almacena conexiones disponibles) + Lock (protege el deque). Se pre-calientan la mitad de las conexiones al iniciar.

Al adquirir una conexión: verifica que esté viva con ping(reconnect=True). Si murió, crea una nueva. Luego hace yield conn al endpoint, y al salir: commit() si todo fue bien, rollback() si hubo excepción.

DictCursor: todas las queries devuelven dict en lugar de tuplas, por eso el código usa row["campo"] en vez de row[0].

get_db() es el FastAPI Dependency que los routers usan con Depends(get_db).

📄 core/auth.py — Validación JWT Azure AD

Pasos de validación de un Bearer token:

  1. Extrae el token del header Authorization: Bearer <token>
  2. Obtiene las claves públicas de Azure AD desde https://login.microsoftonline.com/{TENANT_ID}/discovery/v2.0/keys
  3. Cachea esas claves por 24 horas (JWKS cache)
  4. Decodifica el JWT con python-jose, algoritmo RS256, verificando audience = ClientID
  5. Si falla por rotación de llaves → invalida cache JWKS y lanza HTTP 401

Retorna el payload del token (claims: email, nombre, etc.)

📄 core/permissions.py — Dependencies de permisos

get_current_user(): combina el token JWT con datos de BD (empleado + is_oga) y retorna un objeto CurrentUser completo. Usa cache TTL 5min.

require_oga_user(): llama a get_current_user() y verifica is_oga=True. Si no → HTTP 403 "Esta operación requiere permisos OGA".

📄 core/cache.py — Sistema de cache TTL

CacheTTLTamaño máx.Qué almacena
_arbol_cache15 min1 entradaÁrbol servidor→base→esquema→tablas
_permisos_cache5 min500 usuariosCurrentUser por email
_terminos_cache10 min2 entradasListas de términos frecuentes
_jwks_cache24 h1 entradaClaves públicas Azure AD (JWKS)

Todos los caches tienen su propio Lock para ser thread-safe. Se pueden invalidar manualmente con funciones como invalidar_arbol_cache().

📄 core/exceptions.py — Excepciones tipadas

ExcepciónHTTPCodeCuándo

Cómo arrancar

cd BackendOgaSuite

# 1. Instalar dependencias
pip install -r requirements.txt

# 2. Ejecutar migración SQL en MySQL (ya ejecutado — NewTables.sql)
mysql -h 172.26.61.35 -P 3323 -u apl_ogasuite -p BG_OGASUITE < migrations/NewTables.sql

# 3. Verificar .env con credenciales reales
cat .env

# 4. Arrancar
uvicorn app.main:app --reload --port 8510

# 5. Verificar
curl http://localhost:8510/auth/health
# {"status":"ok","db_connected":true,"version":"1.0.0"}

# 6. Swagger UI
open http://localhost:8510/docs
Todos los endpoints documentados en Swagger: GET /docs muestra la interfaz interactiva con todos los 42 endpoints, sus parámetros, schemas de request/response y la posibilidad de probarlos con un token Bearer.
